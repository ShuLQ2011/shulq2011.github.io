Java编程思想读书笔记(1)

Java编程思想第一章主要是对象导论，难度比较大，涉及到软件工程的内容，可以跳过的，下面就说一下我的理解吧。

面向对象的基本特征

  1. 万物皆为对象

    在生活中，任何一件东西都可以称为对象。如手机，车，房子等。

  2. 程序是对象的合集，它们通过发消息来告知彼此所要做的

    对象封装了数据和对数据的操作，任何你想做的事都由对象来完成，所以面向对象程序自然就是对象的集合。

  3. 每个对象都有自己的由其他对象所构成的存储

    每个对象都可以由其他对象构成，这些对象在计算机中都是占用内存的，被构成的对象的内存等于构成它的对象所占的内存”之和”。

  4. 每个对象都拥有其类型

    一组行为和属性相似的对象就可以构成一个类型，它描述了属于它的对象应该具有的行为和属性，类型是抽象的和概念的。

  5. 某一特定类型的所有对象都可以接收同样的消息

    只要某一对象属于某个类型，那么它就具有这个类型的属性和行为。

更简单的描述： 对象具有状态、行为和标识

意味着每一个对象都可以拥有内部数据（它们给出了该对象的状态）和方法（它们产生行为），并且每一个对象都可以唯一地与其他对象区分开来，具体说来，就是每一个对象在内存中都有一个唯一的地址。

每个对象都有一个接口

此处的接口不是Java中的接口，是指对象实例化后的引用。每个对象都可以实例化，实例化后的对象确定了所能发出的请求，所能发出的请求和隐藏的数据构成了对象的实现。

每个对象都提供服务

在开发中将对象想像成”服务提供者”，程序是通过调用对象来向用户提供服务的。好处是 高内聚 就是指对象的本身”组合”的很好， 不会将过多的功能塞在一个对象中。

被隐藏的具体实现

Java的访问控制是很严格的。 public、private、protected 是Java中的访问指定词。

  1. public 表示任何地方都可以使用。
  2. private 只有在类的内部才可以使用。
  3. protected 只有子类和同一个包中的类可以使用。
  4. 默认访问权限 只有在同一个包中可以使用

复用具体实现

代码的复用是面向对象的优点之一。一个类可以由任意数量、任意类型的其他对象组成，在使用这个类的时候，这个概念称为”组合”，组合是动态发生的称为”聚合”。
组合 是拥有关系，如汽车拥有引擎。在程序设计的时候优先考虑”组合”。
